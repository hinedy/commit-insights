## What to build

Wire `commit-insights generate .` end-to-end: extract commits → analyze → render self-contained HTML dashboard with inline Chart.js. XSS escaping for all user content. Pre-computed aggregates only (no raw commit arrays in HTML).

### Pre-requisites (before first RED→GREEN cycle)

- Install `chart.js` as devDependency (vendored UMD bundle inlined at build time)
- Create `scripts/vendor-chartjs.mjs` — prebuild script: imports Chart.js UMD via `node:module`, writes `src/report/templates/chartjs-bundle.generated.ts`
- Add `"prebuild": "node scripts/vendor-chartjs.mjs"` to `package.json` scripts
- Add `src/report/templates/chartjs-bundle.generated.ts` to `.gitignore`

### Section assembly architecture

Sections are pure `(data) => string` functions — independently testable, assembled by a typed `SectionHTML` record:

```typescript
interface SectionHTML {
  header: string;
  metricCards: string;
  charts: string;            // chart container <canvas> wrappers
  topTickets: string;
  recentCommits: string;
  narrative: string;
  footer: string;
}

// dashboard.html.ts exports:
export function assembleDashboard(sections: SectionHTML, chartJs: string, chartInitScript: string): string
```

No `compose()` indirection — `assembleDashboard()` IS the HTML shell template, with typed slots for every section.

### Chart.js integration

- Chart.js UMD bundle devDependency, inlined at build time via `scripts/vendor-chartjs.mjs`
- Configs built as pure factory functions in `chartConfigs.ts` — no DOM dependency, testable in Node
- Chart data injected into `<script>` via `jsonForScript()` — no raw commit arrays reach HTML
- Global defaults in page script: dark-theme colors, system font, responsive
- Chart containers: consistent `height: 280px` via CSS

### Clockwork

`generate.ts` is wired to the cache system:

```
generate . [--out path] [--no-cache] [--all] [--author val]

  config = loadEffectiveConfig(path)

  if --all:
    commits = extractCommits(path, { all: true, author? })
  elif --no-cache:
    commits = extractCommits(path, { author? })
  else:
    syncResult = syncCache({ repoPath: path })
    commits = syncResult.commits
    // post-filter author + date (cache stores unfiltered)

  areaMap = await mapAreasByFile(commits, config.areas, config.ignorePaths, path)
  analysis = analyzeCommits(commits, config, areaMap)
  dashboardData = buildDashboardData(analysis, commits, repoName)
  narrative = (--narrative && config.ai.provider) ? generateNarrative(provider, stats, opts) : undefined

  renderDashboard({ outputPath, data: { ...dashboardData, narrative }, chartJs })
```

**CLI flags**:
| Flag | Type | Effect |
|------|------|--------|
| `[path]` | positional (default `.`) | repo to analyze |
| `--out` | string (default `dashboard.html`) | output path |
| `--no-cache` | flag | bypass cache |
| `--all` | flag | all branches, bypass cache |
| `--author` | string | filter by author (git `--author` syntax) |

### `DashboardData` interface

```typescript
interface DashboardData {
  repoName: string;
  period: { start: string; end: string };           // "YYYY-MM"
  totals: {
    commits: number;
    tickets: number;                                 // unique ticket count
    authors: number;                                 // unique author count
  };
  timeline: TimelineBucket[];
  typeCounts: Record<string, number>;
  topAreas: Array<{ area: string; count: number }>; // top 10, pre-sorted
  topTickets: Array<{ id: string; count: number }>; // top 15, pre-sorted
  recentCommits: Array<{
    hash: string;
    date: string;
    subject: string;
    type: string;
    area: string;
    tickets: string[];
  }>;                                                // max 200, pre-truncated
  narrative: string | undefined;
  version: string;
  generatedAt: string;                               // ISO timestamp
}
```

A pure function `buildDashboardData(analysis: AnalysisResult, commits: Commit[], repoName: string, narrative?: string): DashboardData` in `src/report/dashboard.html.ts` shapes raw analysis into this display-oriented structure. Render receives `DashboardData` and never touches raw `AnalysisResult` or `Commit[]`.

### Other concerns

- `src/report/escape.ts`: `escapeHtml()` (5-character HTML escape: `& < > " '`) and `jsonForScript()` (escape `</script>`, U+2028, U+2029 in JSON)
- `src/report/charts.ts`: `buildDashboardData()` — pure function, no I/O, testable with fixture arrays
- Chart.js bundled via prebuild script `scripts/vendor-chartjs.mjs`; generated file at `src/report/templates/chartjs-bundle.generated.ts` is gitignored
- No CDN references in default output; `--cdn-charts` opt-in flag considered for future
- **Default output**: CWD (`--out` overrides)
- **Recent commits table**: hard limit of 200 rows, no pagination, sticky header, type badge pills
- **Typography**: system font stack for body; `ui-monospace, "SFMono-Regular", Consolas, monospace` for hashes, ticket IDs, type badges
- **Metric cards**: 3 cards (commits, tickets, authors) — `files` dropped because analysis layer never stores file paths (003 design decision)
- **`authorFilter`** on `extractCommits()` now wired to pass `--author=<val>` to git log (was dead `_authorFilter` placeholder from 002, fixed before 006 cycles)
